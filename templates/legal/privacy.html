Privacidade & LGPD.

Como o CaseHub trata dados de escritórios, clientes e integrações. Versão alpha — 27/05/2026.

Aviso de transparência alpha.

Esta é uma página em construção. O CaseHub alpha (jul/2026 → beta) opera com termos preliminares. Em produção (out/2026+), publicaremos política LGPD completa revisada por advogado especialista. Dúvidas urgentes: victor@vingren.me.

1. Dados que armazenamos

  • Dados do escritório (org): nome, CNPJ, e-mail, telefone, subdomain. Necessários pra prestação do serviço.
  • Dados de clientes/processos/prazos/tarefas cadastrados pelo escritório. Pertencem ao escritório, não ao CaseHub.
  • Tokens OAuth (Google Calendar/Drive): armazenados criptografados em credentials/org_<id>/ no servidor, isolados por escritório.
  • Sessão WhatsApp (whatsapp-web.js): cookies/credenciais do navegador WhatsApp Web, em .wwebjs_auth/session-org-<id>/.
  • Senhas SMTP/IMAP: variáveis de ambiente do servidor, nunca em DB.
  • Backups DB criptografados, retenção 14 dias.

2. Isolamento multi-tenant

Cada escritório opera num tenant separado: subdomain próprio (ex.: sampletenant.casehub.legal), DB filtrado por org_id, tokens OAuth/WhatsApp armazenados em diretórios isolados, queries SQL forçam tenant. Mesmo num servidor compartilhado, dados de escritórios diferentes nunca se cruzam.

Auditoria interna Sentinela (27/05/2026) validou 12 vetores anti-vazamento: T1 (X-Org-Id spoof), T2 (SQL injection), T6 (path traversal), T11 (/uploads público) — todos bloqueados.

3. Direitos do titular (LGPD)

  • Confirmação e acesso aos dados (Art. 18 LGPD): exportação JSON disponível no painel.
  • Correção de dados incorretos: edição direta no app pelo administrador da org.
  • Eliminação de dados: revogação OAuth (Drive/Calendar disconnect), exclusão WhatsApp session, encerramento de org via Suporte.
  • Portabilidade: backup completo SQL fornecido sob demanda.
  • Revogação de consentimento de integrações: botão "Desconectar" em cada integração (Drive, Calendar, WhatsApp).

4. Compartilhamento com terceiros

O CaseHub não compartilha dados de escritórios com terceiros, exceto quando o próprio escritório conecta integrações (Google, Ollama local para Maestro IA). Modelos de IA processam mensagens localmente quando possível; quando a infra usar provedor externo (futuras versões), isso será comunicado e ativado por opt-in.
